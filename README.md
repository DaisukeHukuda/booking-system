# 予約管理システム

「ウラカタ予約」をモデルにした自社専用の予約管理システム。
Cloudflare Workers + Hono + D1 で動作します。

## セットアップ

    npm install
    cp .dev.vars.example .dev.vars   # パスワード等を書き換える
    npm run dev                      # http://localhost:8787

- ヘルスチェック: GET /health
- 管理画面: /admin （.dev.vars の ADMIN_PASSWORD でログイン）

## テスト

    npm test           # 全テスト実行
    npm run typecheck  # 型チェック

## 管理画面の機能

- 予約台帳カレンダー（月表示・空き/満席/連動クローズ/手動クローズの色分け）
- 日別詳細: 空き状況表・予約一覧・新規予約（電話予約の手入力）・変更・キャンセル
- プラン管理: 大人/小人料金・所要時間・リソース割当・時間帯別定員
- 承認待ち: リクエスト予約の承認/否認
- 日別定員の上書き（日別詳細ページから設定）
- 設定: リソース / 時間帯 / 手動クローズ日
- 代理店管理: 専用リンク発行/再発行・予約モード（即時確定/リクエスト）
- 集計: 月次のプラン別・代理店別集計とCSVエクスポート（BOM付きUTF-8・Excel対応）
- 新規予約: 日付自由の手入力（仮予約=リクエスト扱いでの登録も可能）
- 予約検索: 氏名・電話の部分一致、期間・プラン・状態での絞り込み
- 予約台帳: 期間指定の一覧（件数・人数・金額サマリー、CSVリンク）
- 本日の台帳: 時間帯別の当日ロスター（印刷対応）
- 予約枠: 14日マトリクスで日別定員の一括調整とプラン別クローズ/解除

## 代理店ページ

- 専用リンク `/a/{トークン}`（ログイン不要）から空き確認と予約・キャンセル
- 代理店の予約モードにより即時確定またはリクエスト（承認制）
- 予約・承認・否認・キャンセル時にメール通知（Resend。RESEND_API_KEY 未設定時は送信スキップしログのみ）

## 開発ステップ

1. 基盤＋コアロジック — 空き状況計算・アトミック予約登録・管理者認証
2. 管理画面 — 予約台帳カレンダー・予約CRUD・マスタ管理
3. モデル拡張 v2 — 時間重なり連動・日別定員・リクエスト予約・料金内訳
4. 代理店連携 — 代理店マスタ・専用リンク・代理店ページ・メール通知
5. 運用機能（このリポジトリの現状） — 集計・CSVエクスポート・本番デプロイ

## 本番運用

デプロイ先: Cloudflare Workers（無料枠）＋ D1

    npx wrangler deploy                                  # デプロイ
    npx wrangler d1 migrations apply booking-system --remote  # スキーマ反映

シークレット（`npx wrangler secret put <名前>` で設定）:

| 名前 | 必須 | 用途 |
|---|---|---|
| ADMIN_PASSWORD | ✔ | 管理画面のログインパスワード |
| SESSION_SECRET | ✔ | セッションCookieの署名鍵（長いランダム文字列） |
| RESEND_API_KEY | 任意 | メール通知（未設定時は送信スキップ・ログのみ） |
| NOTIFY_EMAIL_TO | 任意 | 通知の宛先メールアドレス |
| NOTIFY_EMAIL_FROM | 任意 | 通知の差出人（Resendで認証したドメイン） |
